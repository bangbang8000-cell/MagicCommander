# MagicCommander 使用指南

## 简介

MagicCommander 是一款专业的网络设备配置管理工具，可以帮助您高效地管理网络设备配置文件，支持 Excel 参数导入、Jinja2 模板渲染、批量输出配置等功能。

---

## 快速开始

### 1. 创建新项目

1. 点击菜单 **文件 → 新建项目**
2. 输入项目名称
3. 点击确认创建

### 2. 配置参数

1. 在项目浏览器中打开 `para.xlsx` 文件
2. 填写设备参数信息（名称、IP、端口等）
3. 保存文件

### 3. 创建模板

1. 在 `templates` 文件夹中创建 Jinja2 模板文件
2. 使用 `{{变量名}}` 语法引用参数

### 4. 执行渲染

1. 切换到 **渲染操作** 面板
2. 选择要渲染的项目
3. 配置输出格式（配置/YAML，是否包含 SN）
4. 点击 **执行渲染**

## 核心功能

### 项目管理

- 创建新项目
- 从模板创建项目（模板中心）
- 打开/删除项目
- 在文件管理器中打开项目目录
- 将项目保存为模板

### 模板中心

模板中心提供内置示例模板，帮助您快速启动项目：

1. 点击左侧活动栏切换到 **模板中心**
2. 浏览可用模板列表（如 ASW/PSW/DOA 交换机配置模板）
3. 点击"从模板创建项目"，输入项目名称
4. 系统自动生成标准目录结构（templates / excel / output / yaml）

您也可以将现有项目保存为模板，供团队复用：
- 右键点击项目 → **另存为模板**

### 参数配置

支持的 Excel 文件格式：
- `.xlsx` - Excel 2007+ 格式
- `.xls` - Excel 97-2003 格式

### 模板渲染

模板语法示例：

```jinja2
# 设备配置模板
system-view
device-name {{device_name}}
ip address {{ip_address}} {{subnet_mask}}
port {{port}}
```

支持的变量：
- `{{变量名}}` - 简单变量
- `{% for item in list %}...{% endfor %}` - 循环
- `{% if condition %}...{% endif %}` - 条件判断

### 渲染预演（Dry-Run）

在正式渲染之前，可以先预览生成结果：

1. 在 **工作台** 面板中，点击项目旁的 **眼睛图标**
2. 系统会渲染模板但不写入文件，实时展示每个设备将生成的配置内容
3. 确认无误后，再点击 **开始渲染** 正式输出

### 模板与数据校验

在渲染之前检查模板和数据质量：

- **Jinja2 模板校验**：点击工作台中的 **校验模板** 按钮，系统会解析所有 `.j2` 模板文件，提前发现语法错误
- **Excel 数据校验**：点击 **校验参数** 按钮，检查 Excel 文件是否存在、sheet 是否为空、列名是否规范
- 校验结果以绿色（通过）/ 黄色（警告）/ 红色（错误）标记

### Diff 对比

对比渲染预演结果与已有输出文件的差异：

1. 执行 dry-run 预演后，在结果中展开设备条目
2. 点击 **Diff 对比** 按钮
3. 系统以统一 diff 格式展示差异（绿色=新增，红色=删除）

### 输出类型

| 类型 | 说明 |
|------|------|
| 配置输出 | 标准配置文件 |
| SN配置 | 带序列号配置 |
| YAML输出 | YAML 格式配置 |
| YAML+SN | YAML 格式带序列号 |

### 搜索与过滤

- **全局搜索**（Ctrl+Shift+F）：支持按文件名、内容搜索项目文件
- **文件类型过滤**：可筛选输出文件（.txt/.cfg）、文本文件（.csv/.json/.html/.py/.log）等
- 搜索结果支持点击跳转到文件编辑

### 标签打印

从设备参数表自动生成设备标签：

1. 切换到 **标签打印** 面板
2. 选择要生成标签的项目
3. 点击 **生成标签**，系统自动输出：
   - Markdown 格式标签（`output-label-md/`）
   - Word 格式标签（`output-label/`）
4. 支持导出 Word（.docx）和 PDF 格式

## 常见问题

### Q: 如何导入设备参数？
A: 在项目根目录编辑 `para.xlsx` 文件，每行代表一个设备，列代表不同参数。

### Q: 模板文件放在哪里？
A: 模板文件应放在项目的 `templates` 文件夹中。

### Q: 输出文件在哪里查看？
A: 渲染后的文件会保存在 `output`、`output-sn`、`yaml`、`yaml-sn` 等文件夹中，可在输出结果面板查看。
